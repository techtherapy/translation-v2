Side Panel Comments — Layout Options

How should a comment panel integrate with the existing 3-column editor?

A

4th Column — Always Visible

Add a dedicated comment column to the right of the translation column. Comments are vertically anchored to their segment. Always visible when toggled on.

┌──────┬────────────┬────────────┬───────────┐
│ ☐ │ Source │ Translation │ Comments │
│   │ 佛陀說法... │ The Buddha │ 💬 "three │
│   │   │ taught... │ hundred"→ │
│   │   │   │ assemblies │
├──────┼────────────┼────────────┼───────────┤
│ ☐ │ Next seg │ Next seg │ (empty) │
└──────┴────────────┴────────────┴───────────┘

Pros

  • All comments visible at once
  • Closest to Google Docs UX
  • Easy to scan for unresolved items

Cons

  • Reduces width of source + translation columns
  • May feel cramped on smaller screens
  • Empty space for segments without comments
B

Slide-Out Panel — On Demand

A panel slides in from the right when you click a comment indicator or the comment button. Overlays or pushes the content. Can be dismissed.

┌──────┬────────────┬────────────┐┌──────────┐
│ ☐ │ Source │ Translation ││ Comments │
│   │   │ The Buddha ││ ━━━━━━━━ │
│   │   │ taught... ││ 💬 Rev.. │
│   │   │   ││ Reply.. │
├──────┼────────────┼────────────┤│   │
│ ☐ │ Next seg │ Next seg │└──────────┘
└──────┴────────────┴────────────┘

Pros

  • Full editor width when not reviewing
  • Focused — shows one segment's comments
  • Familiar slide-out pattern

Cons

  • Can only see one segment's comments at a time
  • Extra click to open/close
  • Overlays may obscure content
C

Margin Annotations — Compact Indicators

Small comment indicators (avatars or icons) appear in the right margin of the translation column, aligned to the highlighted text. Click to expand inline.

┌──────┬────────────┬─────────────────────┐
│ ☐ │ Source │ Translation           │
│   │ 佛陀說法... │ The Buddha taught.. 🔵 │
│   │   │ over three hundred 🔵 │
│   │   │ teachings, and...    │
├──────┼────────────┼─────────────────────┤
│ ☐ │ Next seg │ Next translation      │
└──────┴────────────┴─────────────────────┘

Pros

  • No layout changes — fits current grid
  • Minimal visual noise
  • Combines well with inline highlights

Cons

  • Less space for comment content
  • Still need expand-on-click for full thread
  • Harder to scan across segments
