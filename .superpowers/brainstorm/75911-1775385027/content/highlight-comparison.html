Text-Selection Comments: Two Approaches

How should commented text appear in the editor?

A — Inline Highlights (like Google Docs)
Source: 蓮生活佛

佛陀說法四十九年，講經三百餘會，教化了無數的眾生。

Translation

The Buddha taught the Dharma for forty-nine years, delivered over three hundred teachings, and transformed countless sentient beings.

R
Reviewer 2m ago

"three hundred teachings" → consider "three hundred assemblies" — 會 refers to gatherings/assemblies

HOW IT WORKS
  • Highlighted phrases visible at a glance
  • Click a highlight → opens that comment
  • Highlights disappear when comment is resolved
  • Multiple comments can highlight different phrases
B — Quoted Text in Comment Thread
Source: 蓮生活佛

佛陀說法四十九年，講經三百餘會，教化了無數的眾生。

Translation

The Buddha taught the Dharma for forty-nine years, delivered over three hundred teachings, and transformed countless sentient beings.

R
Reviewer 2m ago
"three hundred teachings"

Consider "three hundred assemblies" — 會 refers to gatherings/assemblies

HOW IT WORKS
  • No changes to the editor text appearance
  • Selected text quoted inside the comment bubble
  • Comment thread looks the same, just with a quote block
  • Much simpler — no TipTap marks or range tracking

Complexity Comparison

A — Inline Highlights B — Quoted Text
Backend changes Add quoted_text + TipTap mark IDs Add quoted_text column only
TipTap integration Custom Mark extension + decorations None — just capture window.getSelection()
Range invalidation Must handle edits shifting/breaking highlights Quote is a snapshot — no tracking needed
Effort estimate Medium-high Low
UX quality Rich — see comments at a glance Good — context is clear in thread
