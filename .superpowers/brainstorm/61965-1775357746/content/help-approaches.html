Help Overlay for Editor — 3 Approaches

How should users discover what each control does?

A

Spotlight Tour

Step-by-step guided tour that dims the page and spotlights each button one at a time. An annotation bubble with a line points to the highlighted control and explains what it does. User clicks "Next" to advance. Like Slack's onboarding.

Pros

  • Focused attention on one thing at a time
  • Can't miss any control
  • Good for first-time users

Cons

  • Slow — must click through every step
  • Annoying for returning users who just forgot one button
  • Complex to implement (positioning, sequencing)
B

Annotated Overlay

A single overlay that dims the background and shows ALL annotations simultaneously — each control gets a numbered callout with a line/arrow connecting it to a brief description. Everything visible at once. User presses "?" or clicks a help icon to toggle.

Pros

  • See everything at a glance
  • Quick to dismiss — one key/click
  • Users can scan for what they need
  • Feels like a "cheat sheet"

Cons

  • Can be visually busy with many controls
  • Need to carefully position annotations to avoid overlap
C

Grouped Cheat Sheet Panel

A slide-out panel or modal listing all controls organized by group (Chapter toolbar, Segment controls, Editing, Track Changes). Each entry shows the icon + label + keyboard shortcut. No visual lines to buttons — just a reference card.

Pros

  • Clean and organized
  • Easy to implement
  • Good as a persistent reference

Cons

  • Doesn't show WHERE buttons are on screen
  • Users still have to match icons mentally
  • Less discoverable / less "wow"

💡 My recommendation: B (Annotated Overlay) — it's the best balance of "show me everything" and quick dismissal. The visual lines connecting labels to buttons are what make it genuinely helpful vs. just another help dialog.
